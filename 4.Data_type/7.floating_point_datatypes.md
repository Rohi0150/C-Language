# Floating Point Datatypes in C

Complete overview of floating point datatypes covering precision, binary representation, and IEEE 754 standard.

## 1. The Comparison Paradox

**Example A:**
```c
float f = 0.6;
if (f == 0.6) printf("hello"); else printf("high");
// Output: high
```

**Example B:**
```c
float f = 0.125;
if (f == 0.125) printf("hello"); else printf("high");  
// Output: hello
```

## 2. Default Type: Double vs. Float

**Key Rule:** Any decimal number is **double by default** in C.

| Feature | Float | Double |
|---------|-------|--------|
| Size | 4 bytes (32 bits) | 8 bytes (64 bits) |
| Precision | Single (23-bit mantissa) | Double (52-bit mantissa) |
| Suffix | f or F (0.6f) | None needed (0.6) |

## 3. Binary Conversion

**Finite (0.125):**
```
0.125 × 2 = 0.25 → 0
0.25 × 2 = 0.5 → 0
0.5 × 2 = 1.0 → 1 (STOP)
Result: 0.001 (3 bits)
```

**Infinite (0.6):**
```
0.6 × 2 = 1.2 → 1
0.2 × 2 = 0.4 → 0
0.4 × 2 = 0.8 → 0
0.8 × 2 = 1.6 → 1
0.6 × 2... (repeats infinitely)
Result: 0.10011001... (infinite)
```

## 4. IEEE 754 Standard

**Float (32-bit):** Sign(1) + Exponent(8) + Mantissa(23)
**Double (64-bit):** Sign(1) + Exponent(11) + Mantissa(52)

## 5. Precision Loss Explained

When assigning double to float, **29 bits are lost** (52 - 23 = 29).

- **0.125:** Discarded bits were zeros → No data loss → Comparison works
- **0.6:** Discarded bits contained ones → Data lost → Comparison fails

**Golden Rule:** Never compare float with double constant!

## Best Practices

✅ **DO:**
```c
float f = 0.6f;
if (f == 0.6f)  // Both float
```

❌ **DON'T:**
```c
float f = 0.6;
if (f == 0.6)  // float vs double
```
