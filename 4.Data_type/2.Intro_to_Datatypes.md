# Intro to Data Types

> A complete guide covering binary addition, variable defaults in C, and the mechanics of signed and unsigned data types.

---

## 1. Binary Addition

Binary addition follows specific rules for **carries**, very similar to how decimal addition works — you sum a column and convert the result into binary to find the **result bit** and the **carry**.

### Key Rules

| Operation | Decimal Result | Binary Result | Result Bit | Carry |
|-----------|---------------|---------------|------------|-------|
| 1 + 1 | 2 | `10` | `0` | `1` |
| 1 + 1 + 1 | 3 | `11` | `1` | `1` |
| 1 + 1 + 1 + 1 | 4 | `100` | `0` | `10` (decimal 2) |

### Comparison to Decimal Addition

In **decimal**, if a column sums to `357`, you write `7` and carry `35` to the next column (tens place).

In **binary**, a carry of `10` means you are carrying the value **2** to the next bit position — it works exactly the same way, just in base-2 instead of base-10.

## 2. Default Nature of Variables in C

When a programmer writes a simple declaration like:

```c
int i;
```

The compiler **internally** interprets this with several implicit/default specifiers:

```c
auto short signed int i;
```

### Implicit Defaults

If not explicitly specified, a variable is implicitly:
- `auto` — storage class
- `short` — size modifier
- `signed` — sign modifier

### Specifier Categories

| Category | Specifiers |
|----------|-----------|
| **Storage Class Specifiers** | `auto`, `static`, `extern`, `register` |
| **Type Modifiers — Size** | `short`, `long` |
| **Type Modifiers — Sign** | `signed`, `unsigned` |

## 3. MSB and LSB

Understanding bit positioning is critical when working with data types.

- **MSB (Most Significant Bit):** The bit that holds the **highest power** (highest positional value).
- **LSB (Least Significant Bit):** The bit that holds the **lowest power** (lowest positional value).

> ⚠️ **Important:** MSB does **not** strictly mean the "leftmost" bit. It is defined by its **power/weight**, not its physical position.

---

## 4. The Signed Data Type

When a variable is declared as **signed**, the compiler treats the **MSB as negative**.

### Example: 4-bit Signed Integer

The bit weights for a 4-bit signed integer are:

| Bit Position | Weight |
|-------------|--------|
| MSB (bit 3) | −2³ = **−8** |
| Bit 2 | 2² = **4** |
| Bit 1 | 2¹ = **2** |
| Bit 0 (LSB) | 2⁰ = **1** |

### Binary to Decimal Mapping (4-bit Signed)

| Binary | Calculation | Decimal |
|--------|------------|---------|
| `0000` | 0 | **0** |
| `0111` | 4 + 2 + 1 | **7** ← Maximum |
| `1000` | −8 | **−8** ← Minimum |
| `1001` | −8 + 1 | **−7** |
| `1111` | −8 + 4 + 2 + 1 | **−1** |

### General Properties of Signed Types

- **Range:** −2^(n−1) to 2^(n−1) − 1
- **MSB = 0** → Number is **positive**
- **MSB = 1** → Number is **negative**
- **Maximum Binary Pattern:** `0` followed by all `1`s → e.g., `0111`
- **Minimum Binary Pattern:** `1` followed by all `0`s → e.g., `1000`
- **−1 in Binary:** All `1`s → e.g., `1111`

---

## 5. The Unsigned Data Type

When a variable is declared as **unsigned**, the compiler treats the **MSB as positive** (no negative weight).

### Example: 4-bit Unsigned Integer

The bit weights for a 4-bit unsigned integer are:

| Bit Position | Weight |
|-------------|--------|
| Bit 3 (MSB) | 2³ = **8** |
| Bit 2 | 2² = **4** |
| Bit 1 | 2¹ = **2** |
| Bit 0 (LSB) | 2⁰ = **1** |

### Binary to Decimal Mapping (4-bit Unsigned)

| Binary | Calculation | Decimal |
|--------|------------|---------|
| `0000` | 0 | **0** ← Minimum |
| `1111` | 8 + 4 + 2 + 1 | **15** ← Maximum |

### General Properties of Unsigned Types

- **Range:** 0 to 2^n − 1
- **All numbers are positive** because every bit weight is positive.
- **Maximum Binary Pattern:** All `1`s → e.g., `1111`
- **Minimum Binary Pattern:** All `0`s → e.g., `0000`

### When to Use Which?

| Use `unsigned` when... | Use `signed` when... |
|------------------------|----------------------|
| Values are **always positive** (e.g., bank balance, array index, memory size) | Values can be **negative** (e.g., temperature, profit/loss) |

---

## 6. Summary Comparison Table

| Feature | Signed | Unsigned |
|---------|--------|----------|
| **MSB Value** | Negative | Positive |
| **General Range** | −2^(n−1) to 2^(n−1) − 1 | 0 to 2^n − 1 |
| **Minimum Binary** | `1` followed by all `0`s | All `0`s |
| **Maximum Binary** | `0` followed by all `1`s | All `1`s |
| **All `1`s Value** | **−1** | **Maximum Value** |

## 7. Important Powers of 2 to Remember

These values are frequently used when calculating data type limits (ranges):

| Power | Value |
|-------|-------|
| 2⁷ | **128** |
| 2⁸ | **256** |
| 2⁹ | **512** |
| 2¹⁵ | **32,768** |
| 2¹⁶ | **65,536** |
| 2³¹ | **2,147,483,648** |

### Applying These to Common C Data Types

For example, using these powers:

- `short int` (16-bit signed): Range = −2¹⁵ to 2¹⁵ − 1 = **−32,768 to 32,767**
- `unsigned short int` (16-bit unsigned): Range = 0 to 2¹⁶ − 1 = **0 to 65,535**
- `int` (32-bit signed): Range = −2³¹ to 2³¹ − 1 = **−2,147,483,648 to 2,147,483,647**
