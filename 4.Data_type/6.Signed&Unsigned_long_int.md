# Signed & Unsigned Long Int

## 1. Signed Long Integer

In C, the `long` keyword is used to specify a larger integer size.

**Default Nature:**
- If you do not specify `signed` or `unsigned`, the compiler defaults to signed.

**Terminology:**
- The following declarations are identical: `signed long int`, `signed long`, and `long`
- In these, `long` is compulsory, while `signed` and `int` are optional.

**Size:** 32 bits (4 bytes)

**Range:** -2,147,483,648 to 2,147,483,647

**Format Specifiers:** 
- `%ld` (long decimal) 
- `%li` (long integer)

---

## 2. Unsigned Long Integer

Used for large positive values.

**Size:** 32 bits (4 bytes)

**Range:** 0 to 4,294,967,295 (0 to 2³²−1)

**Format Specifier:** `%lu` (or sometimes `%ul` depending on the compiler)

*Note: While the code suffixes can be uppercase, the format specifier in `printf` should be lowercase.*

---

## 3. Constants and Suffixes

The compiler treats numeric constants differently based on their value and suffixes:

### Suffixes:
- `L` or `l`: Specifies a `long` constant (e.g., `5L`, `0xFL`)
- `U` or `u`: Specifies an `unsigned` constant
- `LU`, `UL`, `lu`, or `ul`: Specifies an `unsigned long` constant (e.g., `100UL`, `5LU`)

### Automatic Promotion:
- If a number exceeds the range of a short integer (-32,768 to 32,767), the compiler automatically treats it as a `long` integer even without the `L` suffix
- Example: `40,000` is automatically `long`

### The Unsigned Short Trap:
- A value like `40,000U` might be treated as an `unsigned short` because it fits within the 65,535 limit of a 16-bit unsigned short
- To ensure it is treated as an `unsigned long`, use `40,000UL`

## 4. Compiler Architectures & "Plain Int"

The size of a standard `int` (often called "plain int") is not fixed; it depends on the compiler's architecture.

| Data Type | Turbo C (16-bit / DOS) | GCC (32-bit / Linux) |
|-----------|------------------------|----------------------|
| short int | 2 bytes (Always) | 2 bytes (Always) |
| long int  | 4 bytes (Always) | 4 bytes (Always) |
| int (Plain) | 2 bytes (Matches arch) | 4 bytes (Matches arch) |

### Key Points:
- **Rule:** The size of a plain `int` is generally the same as the compiler's bit length (machine cycle capacity)
- **GCC (GNU C Compiler):** Stands for "Genuinely Not Unix" C Compiler
- **64-bit Compilers:** Even in 64-bit versions of GCC, the `int` size is often restricted to 4 bytes to maintain compatibility with languages like Java

---

## 5. Modern Extensions: Long Long

To handle even larger data (8 bytes/64 bits), C introduced `long long`.

**Types:** 
- `long long` (signed) 
- `long long unsigned` (or `unsigned long long`)

**Size:** 8 bytes

**Suffixes:** 
- `LL` or `ll` for signed (e.g., `5LL`)
- `LLU` or `ull` for unsigned (e.g., `0xFULL`)

**Applications:** Used in TCP/IP server communication, encryption, and the CAN protocol in automotive electronics.

---

## 6. Type Casting (Type Conversion)

Type casting is the process of converting data from one type to another.

### Two Types of Casting:

**1. Implicit Type Casting:**
- Performed automatically by the compiler
- Example: Assigning a float `3.5` to an integer variable `a` results in `a = 3`
- "Implicit" means "not clear" in the code

**2. Explicit Type Casting:**
- Performed by the programmer using the cast operator
- Example: `(float)total / 3`
- "Explicit" means "clear"

### Key Rules:
- **Temporary:** Type casting is temporary and only affects the specific statement where it is used
- **Priority:** The type cast operator has higher priority than arithmetic operators (multiplication, division, modulus)
- **Evaluation:** As a unary operator, it evaluates from right to left

---

## 7. Practical Comparison Examples

The behavior of code can change based on the compiler's `int` size.

### Example 1: 32767 + 1

**16-bit Compiler:**
- `int` is short
- Overflow occurs, resulting in `-32768` (Butterfly diagram)

**32-bit Compiler:**
- `int` is long
- Result is `32768` because it is still within the 32-bit range

### Example 2: 200 × 200 (40,000)

**16-bit Compiler:**
- Results in a negative value because 40,000 exceeds the signed 16-bit limit

**32-bit Compiler:**
- Results in `40,000` because it fits in a 32-bit `int`

## Quick Reference Code Examples

### Signed Long Integer
```c
signed long int num1 = -2147483648;  // minimum value
signed long num2 = 2147483647;       // maximum value
long num3 = 100000L;                 // using L suffix
printf("%ld\n", num1);               // using %ld format specifier
```

### Unsigned Long Integer
```c
unsigned long int num1 = 0;          // minimum value
unsigned long num2 = 4294967295UL;   // maximum value with UL suffix
unsigned long num3 = 100000UL;       // using UL suffix
printf("%lu\n", num1);               // using %lu format specifier
```

### Type Casting Example
```c
int a = 10, b = 3;
float result1 = (float)a / b;        // explicit casting: 3.333...
float result2 = a / b;               // implicit casting: 3.0
printf("%f\n", result1);             // prints 3.333...
printf("%f\n", result2);             // prints 3.000...
```
