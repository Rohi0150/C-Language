# Signed Character Data Type in C

> A complete guide covering the `signed char` data type — its size, range, ASCII mapping, internal storage, format specifiers, and overflow/wrap-around behavior.

---

## 1. Introduction to Character Data Type

The character data type in C is represented using the keyword **`char`**.

If you do **not** explicitly write `signed` or `unsigned` before `char`, the compiler **implicitly treats it as a signed character** by default.

### Syntax Variations (All Equivalent)

```c
char ch;
signed char ch;
char signed ch;   // 'signed' can be placed before OR after 'char'
```

All three declarations above are identical in meaning. The compiler accepts `signed` in either position relative to `char`.

---

## 2. Size and Range

- **Size:** 8 bits (1 byte)
- **MSB (Most Significant Bit):** In a signed type, the MSB represents the **sign** (negative weight), splitting the range into half positive and half negative values.

### Range Calculation

The formula for signed data types is:

```
−2^(n−1)  to  2^(n−1) − 1
```

Where `n` = number of bits = **8** for `char`.

```
−2^(8−1)  to  2^(8−1) − 1
−2^7      to  2^7 − 1
−128      to  127
```

### Final Range: **−128 to 127**

| Boundary | Binary Pattern | Decimal |
|----------|---------------|---------|
| Minimum | `10000000` | **−128** |
| Maximum | `01111111` | **127** |
| −1 | `11111111` | **−1** |
| 0 | `00000000` | **0** |

---

## 3. Character Constants and Format Specifiers

### Character Constant

- Any content written within a pair of **single quotes** is called a character constant.
- Examples: `'A'`, `'z'`, `'5'`, `'@'`
- **Rule:** A character constant must have a length of **exactly one** character. Writing more than one character inside single quotes is **not allowed**.

```c
char ch = 'A';   // ✅ Valid
char ch = 'AB';  // ❌ Invalid — more than one character
```

### Format Specifier

- The primary format specifier for printing a character is **`%c`**.

```c
char ch = 'A';
printf("%c", ch);   // Output: A
printf("%d", ch);   // Output: 65 (ASCII value)
```

---

## 4. ASCII Standards

Since computers only understand **binary data (0s and 1s)**, the **ASCII (American Standard Code for Information Interchange)** committee established a universal standard that maps characters to numeric values.

This mapping allows the system to store and interpret characters as numbers internally.

### ASCII Value Table

| Character(s) | ASCII Value(s) |
|-------------|---------------|
| `'A'` – `'Z'` | 65 – 90 |
| `'a'` – `'z'` | 97 – 122 |
| `'a'` | 97 |
| `'b'` | 98 |
| `'d'` | 100 |

> The difference between uppercase and lowercase ASCII values is always **32**. For example, `'A'` = 65 and `'a'` = 97 (97 − 65 = 32).

---

## 5. Data Assignment Logic

In C, you can assign **different types of constants** (character, integer, or float) to a `char` variable, and the compiler handles all internal conversions automatically.

### Integer Assignment

```c
char ch = 97;
printf("%c", ch);   // Output: a
```
The integer `97` is stored as its corresponding ASCII character `'a'`.

### Float Assignment

```c
char ch = 100.75;
printf("%c", ch);   // Output: d
```
The compiler first **truncates** the float `100.75` to the integer `100`, and then stores the ASCII character `'d'` (ASCII 100).

### Key Takeaway

> Regardless of what type of constant you assign — character, integer, or float — the **end result is always stored as a character** because the variable `ch` is declared as type `char`.

| Assigned Value | Internal Conversion | Stored As |
|---------------|---------------------|-----------|
| `'a'` | — | `'a'` (ASCII 97) |
| `97` | Integer → char | `'a'` (ASCII 97) |
| `100.75` | Float → int (`100`) → char | `'d'` (ASCII 100) |

## 6. Internal Storage and Formatting

Understanding **how data is stored internally** is essential for predicting output and avoiding garbage values.

### Storage Forms

| Data Type | Internal Storage Format |
|-----------|------------------------|
| `char` | Direct **binary** |
| `int` | Direct **binary** |
| `float` | **IEEE 754** floating-point standard (NOT direct binary) |

### Format Specifier Behavior

Format specifiers are essentially **data printers** — they read the raw stored data and print it in a specific format.

| Format Specifier | Reads | Prints As |
|-----------------|-------|-----------|
| `%c` | Binary data | Character |
| `%d` or `%i` | Binary data | Signed decimal integer |
| `%o` | Binary data | Octal |
| `%x` or `%X` | Binary data | Hexadecimal (lower/uppercase) |
| `%f` | **IEEE 754** data | Decimal floating-point |

### Garbage Values — The Critical Rule

| Situation | Result |
|-----------|--------|
| Using `%f` to print a `char` or `int` (binary data) | ❌ **Garbage value** — `%f` expects IEEE 754, not binary |
| Using `%c` or `%d` to print a `float` (IEEE 754 data) | ❌ **Garbage value** — these specifiers cannot interpret IEEE 754 |

> ⚠️ **Key Note:** Format specifiers do **NOT** convert data types. They only **format/display** the existing stored data from one representation to another. Using the wrong specifier for the wrong storage format will always produce garbage output.

## 7. Range Overflow — The Butterfly Diagram

When a value assigned to a `signed char` **exceeds its valid range (−128 to 127)**, the value **wraps around** to the other end. This wrapping behavior is often illustrated as the **Butterfly Diagram**.

### Concept

Think of the range as a circular number line:

```
... → 125 → 126 → 127 → [WRAP] → -128 → -127 → -126 → ...
... → -126 → -127 → -128 → [WRAP] → 127 → 126 → 125 → ...
```

---

### Positive Overflow Examples

#### Assigning `128`

128 is **one beyond the maximum (127)**, i.e., `127 + 1`:

```
Binary of  127:   0 1 1 1 1 1 1 1
Binary of    1:   0 0 0 0 0 0 0 1
              + ─────────────────
Result      :   1 0 0 0 0 0 0 0  =  -128
```

`10000000` in signed binary = **−128**

| Assigned Value | Wrapped Result |
|---------------|---------------|
| `128` | `−128` |
| `129` | `−127` |
| `130` | `−126` |

---

### Negative Overflow Examples

#### Assigning `−129`

−129 is **one beyond the minimum (−128)**, i.e., `−128 + (−1)`:

```
Binary of -128:   1 0 0 0 0 0 0 0
Binary of   -1:   1 1 1 1 1 1 1 1
             + ──────────────────
Result      :  [1] 0 1 1 1 1 1 1 1
```

The 9th bit (the carry) is **discarded** because `char` is only 8 bits wide.
Remaining 8 bits: `01111111` = **127**

| Assigned Value | Wrapped Result |
|---------------|---------------|
| `−129` | `127` |
| `−130` | `126` |

---

### Butterfly Diagram Summary

```
        Positive Direction Overflow
               ┌──────────────┐
               ↓              │
  -128 ... -1  0  1 ... 127   │
  │                        └──┘
  └──────────────────────────────────┐
        Negative Direction Overflow  │
                                     ↓
                                   127 ... wraps back to -128
```

| Overflow Direction | Crossing Point | Wraps To |
|-------------------|---------------|----------|
| Positive (exceeds 127) | Above 127 | Starts from **−128** going up |
| Negative (below −128) | Below −128 | Starts from **127** going down |

> **Rule:** Crossing the maximum in the positive direction wraps to the **minimum (−128)**. Crossing the minimum in the negative direction wraps to the **maximum (127)**.
