# Extern Storage Class and Global Variables

This document provides a comprehensive overview of the extern storage class and global variables in C, covering their properties, usage, declaration vs definition, and practical programs.

---

## 1. Introduction to Global Variables

While **static variables** allow the same function to share a variable across multiple calls, **global variables** allow multiple different functions (like `main`, `abc`, and `show`) to share a single variable.

### Properties of Global Variables

| Property | Description |
|----------|-------------|
| **Definition** | Any variable declared **outside of curly braces** (outside of any function body) |
| **Default Value** | Zero (0) |
| **Initialization** | Created and initialized **before** the `main` function starts |
| **Lifecycle** | Exist from the start of the program until the program terminates |
| **Scope** | Accessible from any function in the file (after declaration) |

### Example

```c
int G;  // Global variable (default value = 0)

void test() {
    printf("%d", G);  // Can access G
}

int main() {
    printf("%d", G);  // Can access G
    return 0;
}
```

**Output**: `00`

---

## 2. How Functions Access Variables

When a function encounters a variable (e.g., `G`), it follows a specific **search order**:

### Variable Search Order

1. **Local Check**: The compiler first looks for a **local variable** named `G` within that specific function.
2. **Global Check**: If no local variable exists, the compiler looks for a **global variable** named `G`.

### Visual Representation

```
Function encounters variable G
         ↓
   Is there a local G?
         ↓
    Yes ────→ Use local G
         ↓ No
   Is there a global G?
         ↓
    Yes ────→ Use global G
         ↓ No
    ERROR: Undefined symbol
```

---

### Impact of Changes

Because there is **only one global variable in memory**, if one function updates its value, **all other functions see the updated value**.

### Example

```c
int G = 10;  // Global variable

void increment() {
    G++;  // Updates global G to 11
}

int main() {
    printf("Before: %d\n", G);  // 10
    increment();
    printf("After: %d\n", G);   // 11 (updated by increment())
    return 0;
}
```

**Output**:
```
Before: 10
After: 11
```

---

## 3. The Compilation Problem: "Undefined Symbol"

C compilers process code **from top to bottom**.

### The Problem

If you attempt to use a global variable in a function, but the variable is **defined further down** in the file, the compiler will not have information about it yet.

**Result**: Compilation error: **"Undefined symbol"**

### Error Example

```c
void test() {
    printf("%d", G);  // ERROR: Undefined symbol 'G'
}

int G = 10;  // Defined AFTER usage

int main() {
    test();
    return 0;
}
```

---

### Important Note

A global variable does **not** have to be at the top of the file to be "global," but it **must be known to the compiler before it is used** in a statement.

---

## 4. Definition vs. Declaration

To solve compilation errors without moving code around, you must understand the difference between these two terms:

### Comparison Table

| Feature | Definition | Declaration |
|---------|------------|-------------|
| **Purpose** | To **allocate memory** for the variable | To **convince the compiler** that the variable exists elsewhere |
| **Memory** | Memory **is allocated** | **No memory** is allocated |
| **Example** | `int g;` | `extern int g;` |
| **Occurrence** | **Only once** in the program | Can occur **multiple times** |
| **Location** | Where the variable is actually created | Where the variable is referenced |

---

### Visual Comparison

```c
// DEFINITION (allocates memory)
int G = 10;  // Memory allocated for G

// DECLARATION (no memory allocation)
extern int G;  // Tells compiler: "G exists somewhere, trust me"
```

---

## 5. The `extern` Keyword

The keyword **`extern`** literally means **"outside"**. It is used to create a **declaration** for a global variable that is defined elsewhere (either later in the same file or in a different file).

### Role of `extern`

It tells the compiler: **"Don't allocate memory here. Memory is already allocated in the future or outside this scope. Just allow me to use this name"**.

### Syntax

```c
extern int G;  // Declaration (no memory allocation)
```

### Use Case

Use `extern` when the function call/usage is **above** the actual definition of the variable.

---

### Example: Solving "Undefined Symbol"

#### ❌ Without `extern` (Error)

```c
void test() {
    printf("%d", G);  // ERROR: Undefined symbol 'G'
}

int G = 10;  // Defined later

int main() {
    test();
    return 0;
}
```

---

#### ✅ With `extern` (Correct)

```c
void test() {
    extern int G;  // Declaration: "G is defined somewhere"
    printf("%d", G);  // Now compiles successfully
}

int G = 10;  // Definition (actual memory allocation)

int main() {
    test();
    return 0;
}
```

**Output**: `10`

## 6. Comprehensive Execution Trace

Consider a program where a global variable `G` is shared across `main`, `abc`, and a `test` function using `extern`:

### Complete Program

```c
#include <stdio.h>

int G;  // Global variable definition (default value = 0)

void abc() {
    ++G;  // Increment global G
    printf("Inside abc: G = %d\n", G);
}

void test() {
    extern int G;  // Declaration (if G was defined later)
    ++G;  // Increment global G
    printf("Inside test: G = %d\n", G);
}

int main() {
    printf("Initial G = %d\n", G);
    
    ++G;  // Increment global G in main
    printf("After main increment: G = %d\n", G);
    
    abc();   // Call abc
    test();  // Call test
    
    printf("Final G = %d\n", G);
    return 0;
}
```

---

### Step-by-Step Execution

| Step | Action | Value of G | Output |
|------|--------|------------|--------|
| 0 | Program starts, G initialized | 0 | — |
| 1 | `main` starts | 0 | `Initial G = 0` |
| 2 | `++G` in main | 1 | `After main increment: G = 1` |
| 3 | Call `abc()` | 1 | — |
| 4 | `++G` in abc | 2 | `Inside abc: G = 2` |
| 5 | Return to main | 2 | — |
| 6 | Call `test()` | 2 | — |
| 7 | `++G` in test | 3 | `Inside test: G = 3` |
| 8 | Return to main | 3 | — |
| 9 | Print final G | 3 | `Final G = 3` |

---

### Complete Output

```
Initial G = 0
After main increment: G = 1
Inside abc: G = 2
Inside test: G = 3
Final G = 3
```

---

### The Logic

1. **Start**: `G` is created with value **0** before `main` starts.
2. **Main Function**: `main` executes `++G`. Since there is no local `G`, the global `G` becomes **1**.
3. **ABC Function**: `main` calls `abc()`. Inside `abc`, `++G` executes. It uses the global `G` (now 1) and updates it to **2**.
4. **Test Function**: `main` calls `test()`. Inside `test`, an `extern int G;` declaration is used (if `G` was defined at the bottom).
5. **Final Update**: `test` executes `++G`, updating the global value to **3**.
6. **Output**: When `main` prints `G`, the final result is **3**.

## 7. Basic Programs

### Program 1: Simple Global Variable

```c
#include <stdio.h>

int counter = 0;  // Global variable

void increment() {
    counter++;
}

void display() {
    printf("Counter = %d\n", counter);
}

int main() {
    display();      // Counter = 0
    increment();
    display();      // Counter = 1
    increment();
    display();      // Counter = 2
    return 0;
}
```

**Output**:
```
Counter = 0
Counter = 1
Counter = 2
```

---

### Program 2: Local vs Global Variable

```c
#include <stdio.h>

int x = 100;  // Global variable

void test() {
    int x = 50;  // Local variable (shadows global x)
    printf("Local x in test: %d\n", x);
}

int main() {
    printf("Global x in main: %d\n", x);
    test();
    printf("Global x after test: %d\n", x);
    return 0;
}
```

**Output**:
```
Global x in main: 100
Local x in test: 50
Global x after test: 100
```

**Explanation**: Local variable `x` in `test()` **shadows** the global `x`. The global `x` remains unchanged.

---

### Program 3: Using `extern` Across Functions

```c
#include <stdio.h>

void display() {
    extern int num;  // Declaration: num is defined elsewhere
    printf("Number = %d\n", num);
}

void modify() {
    extern int num;  // Declaration
    num += 10;
}

int num = 5;  // Definition of global variable

int main() {
    display();   // Number = 5
    modify();
    display();   // Number = 15
    return 0;
}
```

**Output**:
```
Number = 5
Number = 15
```

### Program 4: Multiple File Example (Conceptual)

**File 1: main.c**
```c
#include <stdio.h>

extern int sharedValue;  // Declaration (defined in file2.c)

void display();  // Function prototype

int main() {
    printf("Main: sharedValue = %d\n", sharedValue);
    sharedValue = 100;
    display();
    return 0;
}
```

**File 2: helper.c**
```c
#include <stdio.h>

int sharedValue = 50;  // Definition (memory allocated here)

void display() {
    printf("Helper: sharedValue = %d\n", sharedValue);
}
```

**Compilation**: `gcc main.c helper.c -o program`

**Output**:
```
Main: sharedValue = 50
Helper: sharedValue = 100
```

### Program 5: Counter with Multiple Functions

```c
#include <stdio.h>

int totalCalls = 0;  // Global counter

void functionA() {
    totalCalls++;
    printf("Function A called. Total calls: %d\n", totalCalls);
}

void functionB() {
    totalCalls++;
    printf("Function B called. Total calls: %d\n", totalCalls);
}

void functionC() {
    totalCalls++;
    printf("Function C called. Total calls: %d\n", totalCalls);
}

int main() {
    functionA();  // Total: 1
    functionB();  // Total: 2
    functionA();  // Total: 3
    functionC();  // Total: 4
    functionB();  // Total: 5
    
    printf("\nTotal function calls: %d\n", totalCalls);
    return 0;
}
```

**Output**:
```
Function A called. Total calls: 1
Function B called. Total calls: 2
Function A called. Total calls: 3
Function C called. Total calls: 4
Function B called. Total calls: 5

Total function calls: 5
```

## 8. Key Conclusions

### Purpose of Declaration

To **convince the compiler** to allow compilation when the definition is not yet visible.

### Purpose of Definition

To actually **reserve the space in memory** for the data.

### Global Variable Benefit

It acts as a **shared resource** for the entire program, ensuring all functions work with the latest, most accurate data.

---

## 9. Best Practices

### ✅ DO

- Use global variables for data that truly needs to be shared across many functions
- Use `extern` when accessing global variables defined in other files
- Initialize global variables explicitly for clarity
- Use meaningful names for global variables (e.g., `g_totalUsers` with `g_` prefix)

### ❌ DON'T

- Overuse global variables (can lead to hard-to-debug code)
- Use global variables when local or static would suffice
- Forget that global variables are shared (all functions see changes)
- Define the same global variable in multiple files (use `extern` instead)

---

## 10. Summary Table

| Aspect | Global Variable | Local Variable | Static Variable |
|--------|-----------------|----------------|-----------------|
| **Location** | Outside all functions | Inside a function | Inside a function |
| **Scope** | Entire file (after declaration) | Within function only | Within function only |
| **Lifetime** | Entire program | Function call | Entire program |
| **Default Value** | 0 | Garbage | 0 |
| **Memory** | Data segment | Stack | Data segment |
| **Sharing** | Shared across all functions | Not shared | Shared across calls of same function |

---

## Summary

Understanding global variables and the extern storage class is essential for managing shared data across multiple functions and files in C programming.

**Key Points:**
- Global variables are declared **outside all functions**
- Default value is **zero** (not garbage)
- Use **extern** for declaration (no memory allocation)
- **Definition** allocates memory, **declaration** doesn't
- Compiler searches: **local first**, then **global**
- All functions share the **same global variable** in memory
- Use for truly shared data, avoid overuse
